# 後端啟動教學（Windows）— 讓前端吃到真資料庫

目標：把 Spring Boot 後端跑在 `http://localhost:8080`，前端就會自動從 **MySQL → Spring Boot → Vue** 拿到真資料（真正的前後端分離）。

需要三樣東西：**JDK 17**、**MySQL 8**、**Eclipse（你已經有）**。以下逐步來。

---

## 步驟 0：確認 JDK 17

開「命令提示字元」(cmd) 輸入：

```bat
java -version
```

- 有看到 `version "17"` 或更高 → ✅ 跳到步驟 1。
- 沒有 / 版本太舊 → 到 https://adoptium.net 下載 **Temurin JDK 17 (Windows x64 .msi)**，一直下一步安裝完，重開 cmd 再測一次。

---

## 步驟 1：啟動 MySQL，建立資料庫

你 bootcamp 應該裝過 MySQL。確認 MySQL 服務有在跑（Windows 服務裡的 `MySQL80`，或打開 MySQL Workbench 能連上）。

> 沒有的話到 https://dev.mysql.com/downloads/installer/ 下載 **MySQL Installer**，選 Server + Workbench 安裝，安裝時設定的 root 密碼**請記下來**。

### 用 MySQL Workbench 建立資料（最簡單）

1. 打開 **MySQL Workbench**，連上本機 `localhost:3306`（root）。
2. 點上方 **Open SQL Script**（資料夾圖示），打開：
   `桌面\全端\voyago-fullstack\資料庫\01_schema.sql`
3. 按 **⚡ 閃電執行**。會建立 `voyago` 資料庫和 4 張表。
4. 再打開 `02_seed.sql`，一樣按閃電執行 → 灌入 10 條歐洲行程和示範帳號。

### 或用指令（cmd）

```bat
cd /d "%USERPROFILE%\Desktop\全端\voyago-fullstack\資料庫"
mysql -u root -p < 01_schema.sql
mysql -u root -p < 02_seed.sql
```

---

## 步驟 2：把你的 MySQL 密碼填進後端

用記事本打開：
`桌面\全端\voyago-fullstack\後端\src\main\resources\application.yml`

找到這兩行，把 `root`（冒號後面那個）改成**你 MySQL 的 root 密碼**：

```yaml
    username: ${DB_USER:root}
    password: ${DB_PASSWORD:root}        # ← 把這個 root 改成你的密碼，例如 ${DB_PASSWORD:1234}
```

存檔。

---

## 步驟 3：用 Eclipse 匯入後端專案

1. 打開 **Eclipse**。
2. 上方選單 **File → Import...**
3. 選 **Maven → Existing Maven Projects**，按 Next。
4. **Root Directory** 按 Browse，選到：
   `桌面\全端\voyago-fullstack\後端`
5. 確認下面有勾到 `pom.xml`，按 **Finish**。
6. Eclipse 會自動下載 Spring Boot 相關套件（第一次要幾分鐘，右下角有進度條，**等它跑完**）。

> 確認專案用 JDK 17：對專案按右鍵 → Properties → Java Build Path → Libraries，看到的是 JavaSE-17 就對了。

---

## 步驟 4：執行後端

在 Eclipse 左側展開 `src/main/java` → `com.voyago` → 找到 **`VoyagoApplication.java`**，
對它按右鍵 → **Run As → Java Application**。

下方 Console 出現這樣的字就成功了：

```
Tomcat started on port 8080
Started VoyagoApplication in 3.x seconds
```

---

## 步驟 5：驗證後端有通

瀏覽器開：**http://localhost:8080/api/routes**

看到一大串 JSON（行程資料）就代表後端 + 資料庫都通了 ✅

---

## 步驟 6：前端不用改，重整就好

後端起來後，回到前端 `http://localhost:5173`，**重新整理**。
這時資料就是**真的從 MySQL 經過 Spring Boot 傳來的**，不再是前端內建的示範資料。

試試看：註冊一個新帳號 → 預訂一個行程 → 到「會員中心」看訂單 → 去 MySQL Workbench 查 `member`、`booking` 表，會看到你剛剛新增的資料。這就是完整的前後端分離 🎉

---

## 常見錯誤

| 錯誤訊息 | 原因 / 解法 |
| --- | --- |
| `Access denied for user 'root'` | application.yml 裡的密碼跟 MySQL 不符 → 回步驟 2 改對 |
| `Unknown database 'voyago'` | 還沒跑 01_schema.sql → 回步驟 1 |
| `Communications link failure` | MySQL 服務沒啟動 → 開「服務」啟動 MySQL80 |
| Console 顯示 port 8080 被占用 | 有別的程式占用 8080 → 關掉它，或改 application.yml 的 `server.port` |
| 前端還是示範資料 | 後端沒跑成功（步驟 5 打不開）→ 看 Eclipse Console 紅字 |
